# 🕹️ Simple Doodle Jump

یک بازسازی تمیز و کامل از بازی کلاسیک **Doodle Jump** با زبان **C++17** و کتابخانه‌ی گرافیکی **SFML 2.6**.  
این پروژه به‌عنوان تمرین درس **برنامه‌نویسی پیشرفته (Advanced Programming)** و با تمرکز جدی روی **شی‌گرایی (OOP)**، **چندریختی (Polymorphism)** و اصول **Clean Code** نوشته شده است.

این نسخه شامل **فاز ۱** (گیم‌پلی پایه و سکوها) و **فاز ۲** (هیولاها، شلیک، حفره‌ها، سطوح سختی، صدا، و صفحه‌ی تنظیمات) به‌صورت یکپارچه است.

> بازیکن به‌صورت خودکار می‌پرد؛ شما با کلیدهای **← / →** او را چپ و راست هدایت می‌کنید و با **Space** شلیک می‌کنید و تا جای ممکن بالا می‌روید. افتادن به پایین صفحه، برخورد با هیولا از بغل، یا افتادن در حفره = پایان بازی.

## ✨ ویژگی‌ها

### گیم‌پلی پایه (فاز ۱)
- 🎮 گیم‌پلی کامل Doodle Jump: پرش خودکار، دوربین بی‌نهایت رو به بالا، و امتیاز بر اساس ارتفاع.
- 🧱 سه نوع سکو با رفتار چندریخت (polymorphic):
  - **Normal** — سکوی استاندارد (گاهی همراه با **فنر/Spring** برای پرش خیلی بلند).
  - **Moving** — سکوی متحرک که افقی حرکت می‌کند و از دیوارها برمی‌گردد.
  - **Breakable** — سکوی شکستنی که بعد از یک بار فرود می‌شکند و می‌افتد.
- 🌀 **فنر دو-فریمی** با `setTextureRect` (حالت باز/فشرده).
- 🗺️ تولید تصادفی وزن‌دار سکوها با **تضمین دسترسی‌پذیری**: بازی همیشه قابل‌ادامه است.
- 🏆 ذخیره‌ی دائمی بالاترین امتیاز روی دیسک.

### ویژگی‌های فاز ۲
- 👾 **هیولاها** (دو نوع تصویری تصادفی): افقی گشت می‌زنند و از لبه‌های صفحه برمی‌گردند (مثل سکوی متحرک).
  - **فرود از بالا روی سرِ هیولا → پرش قوی و زنده‌ماندن (دقیقاً مثل یک سکو).**
  - هر برخورد دیگر (از بغل، از پایین، یا در حال بالا رفتن) → پایان بازی.
  - با **گلوله** آسیب می‌بینند و می‌میرند؛ میزان جانشان وابسته به سختی است.
- 🔫 **شلیک:** نگه‌داشتن **Space** بازیکن را به حالت شلیک می‌برد و از نوکِ دماغش گلوله رو به بالا پرتاب می‌کند؛ نرخ شلیک وابسته به سختی است.
- 🕳️ **حفره‌ها** (فقط در سختی Hard): برخورد با حفره یک انیمیشن کوتاهِ «مکش» را اجرا می‌کند که بازیکن را کوچک کرده و به داخل حفره می‌کشد و سپس بازی تمام می‌شود.
- 🎚️ **سطوح سختی Easy / Medium / Hard** که روی جانِ هیولا، سرعت سکوهای متحرک، نرخ شلیک و فعال‌بودن حفره‌ها اثر می‌گذارند.
- ⚙️ **صفحه‌ی تنظیمات:** اسلایدر صدا (۰ تا ۱۰۰، زنده) + انتخاب سطح سختی؛ روی دیسک ذخیره می‌شود.
- 🔊 **صدا:** موسیقی لوپ‌شونده (فقط در منو و تنظیمات) + جلوه‌های صوتی پرش/شلیک/باخت، با یک استخر صدا برای پخش هم‌زمان.
- 🥇 **بالاترین امتیازِ مجزا برای هر سطح سختی** (سه مقدار مستقل).

### آخرین به‌روزرسانی‌ها (بازخورد دستیار استاد)
- ⌨️ **صفحه‌ی شروع** حالا متن راهنمای **«Use Left / Right arrows to move»** را نمایش می‌دهد.
- 🖼️ **صفحه‌ی پایان بازی** فریمِ دقیقِ لحظه‌ی باخت را «منجمد» می‌کند و با یک **پرده‌ی مات (frosted)** کم‌رنگ می‌پوشاند تا **مشخص باشد دقیقاً کجا باختی**، و روی آن «YOU LOST»، امتیاز، بهترین امتیاز و دکمه‌های Restart / Menu قرار می‌گیرند.
- 🎯 **اصلاح دقتِ برخورد با هیولا:** حالا فرود از بالا روی هیولا حتی در سقوط‌های بسیار پرسرعت هم همیشه به‌درستی «پرش» حساب می‌شود و هرگز اشتباهاً مرگبار نمی‌شود.

### رفع اشکال و بهبودهای بازبینی نهایی (فاز ۲)
- 🩹 **اصلاح تعویض تکسچر بازیکن:** تکسچرهای دودل هم‌اندازه نیستند (۹۳×۸۰/۸۴ در برابر ۶۲×۶۰ برای حالت شلیک). حالا هنگام تعویض تکسچر، `textureRect` ریست می‌شود و «پاها» ثابت می‌مانند تا نه تصویر خراب شود و نه فیزیک به‌هم بریزد.
- 🎯 **اصلاح برخورد گلوله:** تصویر گلوله (`Nose.png`) فقط بخش بالای بومِ ۲۸×۱۲۰ را پر می‌کند؛ بقیه شفاف بود و باعث می‌شد گلوله از فاصله‌ی دور «به هیولا بخورد». حالا ناحیه‌ی دیدنی برش می‌خورد تا ظاهر و برخورد دقیقاً یکی باشند.
- 🌀 **فنر فقط با تماس واقعی فعال می‌شود:** فرود روی لبه‌ی سکویی که فنر دارد دیگر پرش فنری نمی‌دهد؛ باید بازه‌ی افقی بازیکن با خود فنر هم‌پوشانی داشته باشد (مثل بازی اصلی).
- 🚧 **جای‌گذاری بدون هم‌پوشانی کامل‌تر:** بررسی «فضای خالی» هنگام تولید هیولا/حفره حالا فنرهای بیرون‌زده از سکوها را هم می‌بیند (با `dynamic_cast` روی سکوی نرمال).
- 👾 **گشت‌زنی محدود هیولا:** هر هیولا فقط در یک راهروی محدود اطراف نقطه‌ی تولیدش گشت می‌زند (و جهت شروعش تصادفی است)، بنابراین هرگز تنها مسیر رسیدن به سکوی بعدی را مسدود نمی‌کند.
- 🔇 **مقاوم‌سازی صدا:** خراب یا گم‌بودن فایل صدا دیگر هرگز باعث کرش نمی‌شود (پخش افکت‌ها در برابر خطا محافظت شد).
- 🕳️ **اصلاح انیمیشن مکش:** اگر هنگام افتادن در حفره کلید شلیک نگه داشته شده بود، حالت شلیک رها می‌شود تا انیمیشن کوچک‌شدن درست دیده شود.

### بهبودهای UI
- 🔘 **اندازه‌ی یکسان دکمه‌ها:** تصاویر دکمه‌ها رزولوشن‌های متفاوتی دارند (۴۰۰×۱۰۰ و ۸۰۰×۲۰۰)؛ قبلاً دکمه‌های Settings و Back دو برابر بقیه بودند. حالا همه با عرض ثابت روی صفحه رسم می‌شوند.
- 🎚️ **برچسب صدا با درصد:** در تنظیمات، مقدار صدا به‌صورت «Volume: 58%» نمایش داده می‌شود و بعد از هر تغییر دوباره وسط‌چین می‌شود (قبلاً با تغییر تعداد ارقام از مرکز خارج می‌شد).
- 🥇 **نشان «NEW BEST!»:** اگر رکورد جدیدی ثبت شود، صفحه‌ی پایان بازی آن را با رنگ تأکیدی جشن می‌گیرد.
- 🔴 **امتیاز خواناتر در HUD:** امتیازِ حین بازی با رنگ تأکیدی رسم می‌شود تا روی صحنه‌ی شلوغ گم نشود.

### زیرساخت و کیفیت کد
- ♻️ مدیریت حافظه‌ی امن و بدون نشتی: اشیایی که از صفحه خارج می‌شوند (سکو، هیولا، حفره، گلوله) حذف و اشیای جدید ساخته می‌شوند.
- 🖥️ چهار صفحه‌ی مجزا: **منو**، **تنظیمات**، **بازی**، **پایان بازی** (State Pattern).
- 🧩 معماری تمیز، ماژولار و مستند، آماده‌ی توسعه‌ی بلندمدت.

## 🧱 معماری و طراحی

پروژه کاملاً شی‌گرا و بر پایه‌ی چندریختی از طریق **اشاره‌گر به کلاس پایه + متدهای virtual** ساخته شده است. هیچ‌جا `switch` روی نوع سکو نداریم؛ رفتار متفاوت هر شیء با **dynamic binding** حل می‌شود.

### سلسله‌مراتب کلاس‌ها

```
Entity  (abstract: update / draw / getBounds)
├── Player
├── Spring
├── Monster            # دشمن گشت‌زن؛ سلامتی وابسته به سختی
├── Bullet             # گلوله؛ position_ به‌عنوان مرکز نگه‌داری می‌شود
├── Hole               # تله‌ی مکشِ ثابت (فقط Hard)
└── Platform  (abstract: onLand / isSupportive / supportVelocity / isExpired)
    ├── NormalPlatform     (می‌تواند یک Spring داشته باشد)
    ├── MovingPlatform     (سرعت افقی وابسته به سختی)
    └── BreakablePlatform

State  (abstract: handleEvent / update / render)
├── MenuState
├── SettingsState
├── PlayState
└── GameOverState
```

کلاس‌های هماهنگ‌کننده و کمکی:

- **`Game`** — ریشه‌ی برنامه: پنجره، حلقه‌ی اصلی، منابع، صدا، تنظیمات، بالاترین امتیازها، و اسنپ‌شاتِ فریمِ باخت.
- **`World`** — هماهنگ‌کننده‌ی دنیا: `PlatformManager` را **کامپوز** می‌کند و هیولاها و حفره‌ها را کنارش مدیریت می‌کند.
- **`CollisionSystem`** — حل برخوردِ متمرکز برای سکوها (`resolve`) و هیولاها (`resolveMonsters`).
- **`PlatformManager`** — تولید/حذف سکوها.
- **`AudioManager`** — موسیقی و جلوه‌های صوتی با استخر صدا.
- **`Settings` / `Difficulty`** — تنظیماتِ ماندگار و سطح سختی.
- **`ResourceManager<T>`** — کش عمومی منابع (`sf::Texture` و `sf::Font`).
- **`Button` / `Slider` / `ui::Layout`** — اجزای رابط کاربری.
- **`Random`** — موتور تصادفِ مشترک.

### تصمیم‌های کلیدی طراحی

- **تک‌مسئولیتی (SRP):** هر کلاس فقط یک کار دارد.
- **برخورد متمرکز:** همه‌ی سکوها از یک منطق برخورد واحد عبور می‌کنند؛ *پاسخِ* متفاوت با `Platform::onLand()` به‌صورت چندریخت واگذار می‌شود. همین تستِ حرکتی (swept) برای هیولاها هم بازاستفاده شده است.
- **تنها مرجعِ عدم‌هم‌پوشانی:** متد `World::areaIsFree()` هر مخاطره‌ی جدید را پیش از ساخت، در برابر **همه‌ی** سکوها/هیولاها/حفره‌ها بررسی می‌کند تا هیچ‌چیز روی چیز دیگری ساخته نشود. مخاطرات هم فقط **کنارِ** یک سکو و پس از یک تأخیر اولیه ساخته می‌شوند تا هرگز تنها مانعِ مسیر امن نباشند.
- **قالب معنادار (Template):** `ResourceManager<T>` برای `sf::Texture`، `sf::Font` و `sf::SoundBuffer` استفاده می‌شود.
- **صفحه‌ی باختِ «منجمد»:** متد مشترکِ `drawScene(target)` هم روی پنجره‌ی زنده و هم روی یک `sf::RenderTexture` خارج از صفحه کار می‌کند؛ در لحظه‌ی باخت، صحنه در این بافت ذخیره و به `Game` سپرده می‌شود تا صفحه‌ی پایان آن را (با پرده‌ی مات) نمایش دهد.
- **RAII و مالکیت امن:** استفاده از `std::unique_ptr` برای State فعال و اجزای UI؛ حذف صریح اشیای خام (`Monster*`, `Hole*`, `Bullet*`) در مالکِ آن‌ها.
- **فیزیک مستقل از نرخ فریم:** همه‌ی حرکت‌ها در `dt` ضرب و `dt` برای پایداری کلمپ می‌شود (جلوگیری از تونل‌کردن).
- **بدون Magic Number:** تمام مقادیر قابل‌تنظیم در `include/core/Constants.hpp` (namespace `cfg`) متمرکز شده‌اند.
- **دوربین:** یک `sf::View` عمودی که فقط رو به بالا حرکت می‌کند.

### تضمین دسترسی‌پذیری (Accessibility)

حداکثر فاصله‌ی عمودی بین دو سکو `MAX_GAP = 95px` است و `2 × 95 = 190px` از حداکثر ارتفاع پرش معمولی (`≈ 225px`) کمتر است. همچنین هیچ‌گاه دو سکوی شکستنی پشت‌سرهم قرار نمی‌گیرند و هیچ مخاطره‌ای روی مسیر امن ساخته نمی‌شود. نتیجه: بازی همواره قابل‌ادامه است.

## 🛠️ پیش‌نیازها

1. کامپایلر **g++** با پشتیبانی **C++17**
2. ابزار **make** (روی ویندوز: `mingw32-make`)
3. کتابخانه‌ی **SFML 2.5 یا 2.6** با ماژول‌های **`graphics`, `window`, `audio`, `system`**

> ⚠️ ماژول **audio** برای صدا الزامی است (فاز ۲). روی ویندوز علاوه بر DLLهای گرافیکی، به `sfml-audio-2.dll` و `openal32.dll` هم نیاز داری.
>
> ⚠️ کد برای **SFML 2.x** نوشته شده است. با **SFML 3** به‌طور مستقیم کامپایل نمی‌شود (API نسخه ۳ متفاوت است).

## ▶️ ساخت و اجرا

### لینوکس (Ubuntu / Debian)
```bash
sudo apt update
sudo apt install g++ make libsfml-dev
make
make run
```

### macOS (Homebrew)
```bash
brew install sfml@2
# Apple Silicon:
make SFML_INC=-I/opt/homebrew/opt/sfml@2/include SFML_LIB=-L/opt/homebrew/opt/sfml@2/lib
./simple-doodle-jump
```

### ویندوز (MinGW-w64 + Git Bash)
1. کامپایلر **GCC** (WinLibs 64-bit) و نسخه‌ی متناظرِ **SFML 2.6.1 برای همان GCC (MinGW SEH 64-bit)** را نصب کن (نسخه‌ی کامپایلر باید ۱۰۰٪ با SFML یکی باشد).
2. در **Git Bash**، از ریشه‌ی پروژه:
```bash
mingw32-make SFML_INC=-IC:/SFML/include SFML_LIB=-LC:/SFML/lib
```
3. فایل‌های `*.dll` داخل `C:\SFML\bin` را کنار `simple-doodle-jump.exe` کپی کن (یا `C:\SFML\bin` را به PATH اضافه کن). حواست باشد **`sfml-audio-2.dll` و `openal32.dll`** را هم کپی کنی.
4. اجرا:
```bash
./simple-doodle-jump.exe
```

> اگر SFML را جای دیگری نصب کرده‌ای (مثلاً `C:\Program Files\SFML-2.6.1`)، به‌خاطر فاصله در مسیر از فرم کوتاه استفاده کن:
> ```bash
> mingw32-make "SFML_INC=-IC:/PROGRA~1/SFML-2.6.1/include" "SFML_LIB=-LC:/PROGRA~1/SFML-2.6.1/lib"
> ```

### دستورهای makefile
| دستور | کار |
|-------|-----|
| `make` | ساخت بازی |
| `make run` | ساخت و اجرا |
| `make clean` | پاک‌کردن خروجی‌های ساخت |

> 📌 بازی را حتماً از **ریشه‌ی پروژه** اجرا کن؛ مسیرهای `assets/`، `sounds/`، `fonts/`، `settings.txt` و `highscore.txt` نسبی هستند.

## 🧼 استاندارد کد و نام‌گذاری

این پروژه از یک گایدلاین نام‌گذاری واحد پیروی می‌کند که در ابتدای هر فایل هم اعلام شده است:

| دسته | قاعده | نمونه |
|------|-------|-------|
| کلاس‌ها / تایپ‌ها | `PascalCase` | `PlatformManager` |
| توابع / متدها | `camelCase` | `handleInput` |
| متغیرها و پارامترها | `lower_snake_case` | `world_top` |
